# JS / TS Exports

> Optional JS/TS API for places where CSS variables don't fit — inline styles, `matchMedia`, chart libraries, SSR. Components do NOT use this — they consume tokens via CSS variable names.

## Token List

`dist/tokens.js` exposes the following named exports:

### Foundation refs (theme-aware, resolve at runtime via `var(...)`)

| Export | Resolves to |
|---|---|
| `colours` | `var(--mds-colours-*)` ref strings, nested. |
| `radius` | `var(--mds-radius-*)`. |
| `spacing` | `var(--mds-spacing-*)`. |
| `strokes` | `var(--mds-strokes-*)`. |
| `grid` | `var(--mds-grid-*)`. |
| `layout` | `var(--mds-layout-*)`. |
| `numberRamp` | `var(--mds-number-ramp-*)`. |
| `commonComponentValues` | `var(--mds-common-component-values-*)`. |
| `typography` | `var(--mds-typography-*-*)`. |

### Build-resolved primitives (not theme-aware)

| Export | Resolves to |
|---|---|
| `palettes` | Numeric/hex primitives from `src/base/`. Resolved at build time. |
| `breakpoints` | Numeric viewport thresholds (`{ mobile: 360, tablet: 768, laptop: 1040, desktop: 1440 }`). |

## Usage

```ts
import { colours, breakpoints, palettes } from '@mds/tokens';

// Foundation tokens are CSS var ref strings (theme-aware at runtime)
<div style={{ background: colours.basic.accent }} />          // → "var(--mds-colours-basic-accent)"

// Numeric primitives (resolved at build, not theme-aware)
window.matchMedia(`(min-width: ${breakpoints.tablet}px)`)     // 768
const successHex = palettes['green light'].colours.basic.accent;
```

### When to use JS exports

- **Inline styles** that need a theme-aware ref string.
- **`matchMedia`** queries for responsive logic.
- **Chart libraries** that take colour as a string prop.
- **SSR** scenarios where CSS var resolution isn't available.

### When NOT to use JS exports

- **Component CSS Modules** — always reference tokens by CSS var name. The `tokens.js` exports are not for component styling. See [[conventions/tokens-policy]].

## TypeScript

`dist/tokens.d.ts` is generated by a custom `ts/foundation-types` format that mirrors the `tokens.js` shape exactly (nested objects, `string` leaves for refs, typed `palettes` and `breakpoints`).

Key naming: camelCase JS-friendly transforms of the kebab-case CSS var paths.

| CSS var | JS key |
|---|---|
| `--mds-colours-basic-text-on-accent` | `colours.basic.textOnAccent` |
| `--mds-common-component-values-height-large` | `commonComponentValues.heightLarge` |
| `--mds-number-ramp-24` | `numberRamp[24]` (numeric key preserved) |

TS strict mode is supported.

## Layer

JS exports are emitted by Style Dictionary's `mds/js` transform group, alongside the CSS chunks. See [[architecture/token-pipeline]].

## Naming Pattern

- camelCase JS-friendly keys derived from kebab-case CSS var paths.
- Numeric keys preserved as numbers (`numberRamp[24]`, not `numberRamp.value24`).
