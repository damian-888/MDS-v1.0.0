# Registry

> Generated registry mapping kebab-case icon names to lazy React components. Currently 1505 icons. The `MDSIconName` literal union gives compile-time autocomplete for every available name.

## Rule

### What the registry exports

[`packages/mds-icons/src/generated/`](packages/mds-icons/src/generated/) is fully generated by `pnpm --filter @mds/icons generate`. It produces:

| File | Contents |
|---|---|
| `<PascalCase>.tsx` (per icon) | One file per icon — a `forwardRef` React component using `currentColor` for fill. 1505 files. |
| `registry.ts` | The `iconRegistry` object — kebab-case keys, `React.lazy()` values. Also emits the `MDSIconName` literal union type. |
| `index.ts` | Barrel exporting `iconRegistry`, `MDSIconName`, `MDSIconProps`. |
| `types.ts` | The `MDSIconProps` shape (`SVGProps<SVGSVGElement>` minus `width`/`height`, plus `title` / `titleId`). |

### What `@mds/icons` exports

The package's barrel re-exports the three public symbols:

| Export | Type | Purpose |
|---|---|---|
| `iconRegistry` | `Record<MDSIconName, LazyExoticComponent<...>>` | The lazy registry. Consumed by `MDSIcon`. |
| `MDSIconName` | string literal union | Every available icon name. Used for autocomplete and type-checking. |
| `MDSIconProps` | TS interface | The base props shape that individual icon components accept. |

`MDSIcon` (the component consumers actually render) lives in `@mds/components`, not `@mds/icons`. See [[components/MDSIcon]].

### Generation

The registry is **re-generated** each time `pnpm --filter @mds/icons generate` runs (which is also a step of `pnpm --filter @mds/icons build`). Hand-editing files in `src/generated/` will be overwritten on the next sync.

To **add** an icon: add it in Figma's icons library, then run `pnpm --filter @mds/icons sync && pnpm --filter @mds/icons build`. See [[workflows/add-icon]].

## Why

- **One source of truth.** Adding an icon requires only a Figma change + a sync; no manual registry edits.
- **Compile-time correctness.** `MDSIconName` is a literal union — typos in `name=""` fail at TypeScript level.
- **Code-splitting.** The registry's values are `React.lazy()` stubs, and the build is configured so each value resolves to its own chunk. See [[lazy-loading]].

## How to Apply

### Looking up an icon by name

```tsx
import { MDSIcon } from '@mds/components';
import type { MDSIconName } from '@mds/icons';

const myIcon: MDSIconName = 'chevron-down';      // typo here = compile error
<MDSIcon name={myIcon} />
```

### Iterating all icons

```ts
import { iconRegistry } from '@mds/icons';
const allNames = Object.keys(iconRegistry) as MDSIconName[];
// → all 1505 kebab-case names
```

This is rarely useful in app code but powers tools (icon search UIs, docs sites). Note: iterating the registry does NOT eagerly load the icons — only `<LazyIcon />` rendering triggers the dynamic import.

## Examples

```ts
// The literal union — autocomplete in IDEs lists every name.
type MDSIconName = 'acorn' | 'address-book' | 'address-book-tabs' | /* 1505 in total */ | 'youtube-logo';
```

```ts
// Inside the registry file (generated):
const iconRegistry = {
  'acorn': lazy(() => import('./Acorn').then(m => ({ default: m.Acorn }))),
  'address-book': lazy(() => import('./AddressBook').then(m => ({ default: m.AddressBook }))),
  // ... 1500+ entries
} as const;
```

## Exceptions

- **Generated files.** Don't hand-edit anything in `src/generated/`. The next `generate` run will overwrite. To customise an icon, modify the SVG source in `src/svg/` (or upstream in Figma) and re-run the pipeline.
- **Custom SVG template.** The generated component shape (forwardRef + `currentColor`) is set by [`scripts/icon-template.mjs`](packages/mds-icons/scripts/icon-template.mjs). Changes to that template apply to all icons on the next `generate`.
